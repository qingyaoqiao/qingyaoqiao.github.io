从零开始，将你的 Node.js + Python 项目打包成 exe 安装包（小白版保姆教程）

如果你有一个和我一样的项目——前端是网页（HTML/JS/CSS），后端主力是 Node.js，但又需要调用 Python 脚本来处理一些特殊任务（比如我的 PDF 转换），并且想把它做成一个能发给别人直接用的 .exe 安装包，那么这个帖子或许可以给你提供一些参考。

前言：核心思路与三大天坑

核心思路： 我们的目标是让用户电脑上即使没有 Node.js 和 Python 环境，也能运行我们的软件。方法是：

  1. 用 PyInstaller 把 Python 脚本和它的依赖库打包成一个独立的 .exe 文件。
  2. 用 Electron 给我们的 Node.js 服务器和前端网页套上一个“壳”，做成一个桌面应用。
  3. 让 Electron 应用去调用那个 Python 打包好的 .exe 来完成任务。

三大天坑预警（非常重要！）：

  1. 中文路径天坑：从你的 Windows 用户名，到项目文件夹路径，任何地方出现中文，都可能导致各种匪夷所思的报错。这是万恶之源！
  2. 网络天坑：Electron 打包时需要从国外服务器下载很大的文件，国内网络环境基本都会失败。必须使用国内镜像。
  3. 权限天坑：打包好的程序默认是“只读”的，不能随便在自己的安装目录里写文件。同时，它也没有权限在系统里创建一些特殊文件（比如软链接）。

第一步：准备工作 - 净化你的开发环境

这是最重要的一步，能避免后续 80% 的问题。

  1. 正确安装 Python：

    • 去 Python 官网下载 64位 的安装包。
    • 安装时，务必勾选 Add python.exe to PATH。
    • 选择“自定义安装”，把安装路径改到一个简单的英文路径，比如 C:\Python311，绝对不要装在包含中文用户名的默认路径下！

  2. 移动你的项目代码：

    • 在 C 盘或 D 盘根目录，新建一个全英文的文件夹，比如 C:\work。
    • 把你的整个项目文件夹（比如我的 ai-translator）剪切到这个新目录里。
    • 最终你的项目路径应该是这样的：C:\work\ai-translator。

第二步：将 Python 脚本“独立化”

这一步的目的是让我们的 Python 脚本摆脱对用户电脑上 Python 环境的依赖。

  1. 安装 PyInstaller：
    打开 PowerShell（或 CMD）命令行工具，进入你的项目目录：

  2. 执行打包：
    假设你要打包的 Python 文件叫 converter.py。

    • --onefile：把所有东西打包成一个单独的 exe 文件。
    • --name converter：给生成的 exe 文件起个名字。

  3. 整理文件：

    • 打包成功后，项目里会出现一个 dist 文件夹。
    • 在你的项目根目录 (C:\work\ai-translator)下，新建一个文件夹，命名为 bin。
    • 把 dist 文件夹里的 converter.exe 复制到 bin 文件夹里。

第三步：创建 Electron 应用外壳与代码改造

现在，我们要开始搭建桌面应用的“架子”，并修改代码来适应打包环境。

  1. 创建 Electron 入口文件：
    在项目根目录新建一个 electron-main.js 文件，这是 Electron 启动的入口。它负责创建窗口、加载我们的网页、隐藏丑陋的默认菜单栏。

  2. 改造 server.js，解决路径和权限问题：
    打包后的程序，不能再用相对路径，也不能随便往自己的安装目录里写文件。所以我们必须修改 server.js，让它使用绝对路径，并把文件上传和下载的临时目录改到系统的 Temp 文件夹。

    (请根据你的业务逻辑，确保所有 fs 操作都基于 WORK_DIR)

  3. 改造 translator.js，让它能“指挥”exe：
    我们需要让 Node.js 知道什么时候该运行 python converter.py（开发时），什么时候该运行 bin/converter.exe（打包后）。

第四步：配置 package.json 并最终打包

这是决战前的最后准备。

  1. 配置 package.json：
    这个文件是打包的总指挥。需要指定入口文件、打包脚本，以及最重要的——告诉打包工具把我们的 bin 文件夹（里面有 converter.exe）也一起带上。

  2. 安装依赖：
    清理旧的依赖，使用国内镜像重新安装，又快又稳。

  3. 执行打包 (决战时刻)：

    • 解决网络问题：在 PowerShell 中设置临时的环境变量，让它去国内镜像下载打包工具。
    • 解决权限问题：如果打包时报“无法创建符号链接 (Cannot create symbolic link)”的错误，需要开启 Windows 的“开发者模式”。
      • 打开 Windows 设置 -> 系统 -> 开发者选项 -> 打开“开发者模式”开关。
    • 开始打包！

结尾

如果一切顺利，你会在项目里看到一个 dist-app 文件夹，里面那个 .exe 安装包就是你的胜利果实。

回顾一下，成功的关键在于：

  1. 保持路径纯净：全程使用英文路径。
  2. 内外有别：代码要能区分开发环境和打包环境，并使用不同的资源路径。
  3. 遵守规则：打包后的程序是“客人”，要使用系统指定的临时目录来读写文件，不要在自己家里乱动。

希望这份教程能帮你把想法变成现实。祝你打包顺利！
