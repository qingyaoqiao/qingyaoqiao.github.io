如遇预以下情况：

  1. 电脑 Windows 用户名是中文的（如 C:\Users\小白\）。
  2. 在使用 hexo d 部署博客时，经常遇到 Connection was reset 或 Permission denied 错误。
  3. 想配置 SSH 密钥，却提示 Could not create directory。
  4. 使用 WebStorm 写博客，希望在终端里一键部署。

这篇文章或许可以给到一些参考（来自本人因为不愿意注销用户名和重装系统，又想用SSH的经验分享）。

问题根源

Windows 的中文用户名在很多开发工具（如 Git、SSH）中会导致路径乱码或无法识别。默认情况下，SSH 试图去 C:\Users\中文名\.ssh 读取密钥，结果找不到路径，导致部署失败。

我们不修改用户名（麻烦、风险、绑定多），而是给 SSH 另外指一条路（D盘贵宾一位~）。

准备工作

  • 已安装 Git
  • 已安装 Node.js 和 Hexo
  • (可选) 使用 WebStorm 作为编辑器（VS Code 同理）

第五步：WebStorm 终端终极配置 (最关键)

为了让 WebStorm 的终端也能识别我们自定义的 SSH 路径，我们需要设置环境变量。

  1. 打开 WebStorm，进入 Settings (设置) -> Tools (工具) -> Terminal (终端)。

  2. 设置 Shell path (Shell 路径)：
    确保这里指向的是 Git 的 bin 目录下的 bash.exe，不要用 usr/bin 下的。

    • 正确示例："D:\Program Files\Git\bin\bash.exe"
    • 请根据你 Git 的实际安装位置修改盘符和路径，记得保留英文双引号。

  3. 设置 Environment variables (环境变量)：
    点击右侧的文件夹图标，添加一个新的变量：

    • Name: GIT_SSH_COMMAND
    • Value: ssh -F /d/.ssh/config

    原理：这个变量告诉 Hexo 和 Git，“每次执行 SSH 命令时，请强制带上 -F /d/.ssh/config 这个参数，去读取我 D 盘的配置文件”。

  1. 保存设置，并关闭所有已打开的终端窗口，重新打开一个新的 Terminal。

总结

不修改系统用户名，而是通过自定义密钥路径 + SSH Config 文件 + IDE 环境变量 的组合连招，完美绕过 Windows 对中文路径支持不友好的坑。
